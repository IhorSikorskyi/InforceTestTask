This is my Test Task for joining Inforce Company

  • About: contains information about my algorithm for short URLs.
  • IUrlShortenerService Interface with advertisement my public method in UrlShortenerService with Injection.
  • UrlShortenerService Class with implementation of IUrlShortenerService. It contains public methods what were injected from Interface and one private - Encode() with Base64 encode algorithm. I also can use method from Convert - Convert.ToBase64String, but for this Task I wrote this algorithm.

    In CreateShortUrlAsync() method I use Encode() to encode long URL and create short URL. Because all URLs must be unique, I check if the short URL already exists in the database. If it does, I generate a new short URL by appending a random number to the encoded string until I find a unique one. After that, I save the new short URL to the database and return it.

    To make short links work correctly I created GetShortUrlInfoAsync() method. It readdresses the short URL to the original long URL.

    I have also implemented GetShortUrlInfoAsync() method to retrieve information about a short URL.

    Then I created a method to remove short URL from the database - DeleteShortUrlAsync().

    In the end I added a method to show information about short URL - GetShortUrlInfoAsync().

    Of course, I changed the middleware to add this DIs to the pipeline.

  • ShortUrlsController Class with methods to handle HTTP requests for short URLs. It includes endpoints for creating, retrieving, deleting, and getting information about short URLs.
  • Unit Tests: I also added some unit tests for the UrlShortenerService and other services. All tests are located in the TestTaskTests project and pasted successfully.
